    はじめに

    ここでは、アプリケーションの設計手法について学びます。

    目的によって大きく下記のコンテンツに分けてみました。学習の順番は特にありません。読みやすいところから始めてもらって大丈夫です。

    • 保守しやすい設計

      1
      「高品質のものをより早く」を実現するには、保守しやすい設計。コーディングが必要です。このコンテンツでは、保守しやすい設計・コーディングの基本的な考え方を解説します。

    • ドメイン駆動設計

      1
2
3
      メンテナンス性の良いアプリケーションの設計を行うための優れたガイドラインで、上流から下流の設計まで網羅するものです。

Webアプリに限定したものではありませんが、ここではWebアプリ開発を例に解説します。

    • UI/UX 設計

      1
      利用者に寄り添った 良い UI/UX を設計するため、理論とパターンを学習します。

    • API 指向 と Rest(ful) API 設計

      1
      原則を理解することで、システム連携が容易な API を提供しましょう。

    • Webアプリケーションのアーキテクチャ

      1
      Webアプリケーションの構築する上のアーキテクチャとプラットフォームの選定のヒントとして関連技術を紹介します。

    • Web アプリケーション の セキュリティ

      1
2
3
      セキュリティとは不正アクセスや攻撃からの防衛だけでなく、設定やロジックの不備、運用のミスによるサービス提供に支障が出るような事故も含まれます。

Web アプリケーションにおけるそれら一般的なセキュリティに関する事項を解説します。
