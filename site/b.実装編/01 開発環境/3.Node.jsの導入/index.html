    Node.js の導入

    概要

    Node.js は、Webサービスなどのサーバーサイドや、Lambdaなどのサーバーレス環境で利用されるJavaScriptのランタイムです。

    動作環境だけでなく、Webアプリ開発の環境としても広く利用されています。

    開発環境としては、後で説明するコンテナを利用することで、ローカルにインストールしなくても開発はできるのですが、バッチで利用するなどなにかと役に立つので、入れておいても損はないです。

    Node.jsとはなにか？なぜみんな使っているのか？ - Qiita

    インストール

    公式サイトや Linux の apt や yum と言ったパッケージマネージャでインストールすることもできますが、コンテナを利用しない開発で、プロジェクトごとに必要なバージョンが異なることもあります。

    ここでは、Node.js のバージョン管理ツール「n」を利用してインストールします。

    n

    「n」は、Node.js のパッケージマネージャ npm を利用してインストールするので、下記手順で行います。

    1. Node.js と npm を仮インストールする。
    2. npm を使って n をインストールする。
    3. n を使って、Node.js をインストールする。
    4. 1. でインストールした Node.js を削除する。

    1. Node.js と npm を仮インストールする

    Ubuntu を前提に説明します。apt を使って、Node.js をインストールします。

    1
2
3
4

# インストールのログが表示される

# インストールされた Node.js のバージョンを確認する
$ node -v
v12.22.9

    sudo は、システム管理者権限でコマンドを実行するためのコマンドです。パスワードが要求されます。WSLをはじめに起動した時に設定したパスワードを入れてください。

    Node.js の安定バージョンは、2023年3月時点で 18.15.0 なのですが、だいぶ古いバージョンが入ったのではないでしょうか。

    これは n をインストールするための仮のものなので、今はこれで良いです。

    4. apt でインストールした Node.js を削除する

    1

    削除が完了したら、環境変数を取得し直すために、ターミナルを一旦閉じて、再度開きます。

    その後で、n でインストールした Node.js のバージョンを確認します。

2

    Ctrl+D で終了します。

    JavaScript のファイルを使って実行する

    Node.js は、JavaScript のファイルを実行してみましょう。

    ここでは、Node.jsの機能を確認するために、ファイルを生成してみます。hello.mjs というファイルを作成し、下記のコードを記述します。

    1
2
3
4
5
    import fs from "fs/promises";

const text = `Hello World! ${new Date().toISOString()}`;

await fs.writeFile("./hello.txt", text, { encoding: "utf-8" });

    ファイルを実行してみましょう。

    1
2
3
4

$ ls

$ more hello.txt
Hello World! 2023-03-17T07:35:58.880Z

    hello.txt というファイルが生成され、その中にプログラムで記述した文字列が書き込まれています。

    JavaScript ファイルをスクリプトとして実行する

    Node.js をシェルとして、ファイルを指定するだけで実行できるようにすることもできます。

    ファイルの1行目に、#!/usr/bin/env node と記述して、ファイルに実行権限を付与します。

    1

import fs from "fs/promises";

const text = `Hello World! ${new Date().toISOString()}`;

await fs.writeFile("./hello.txt", text, { encoding: "utf-8" });

    yarn パッケージマネージャを導入する

    Node.js は、パッケージマネージャとして npm が標準で付属していますが、yarn というサードパーティ制のパッケージマネージャもあります。

    yarn は npm と互換性があり、パッケージを管理する package.json ファイルをそのまま使うことができます。またパッケージ自体も npm のものが使えます。

    yarn を使うメリットとしては、

    • パッケージのインストールが早い
    • コマンドがわかりやすい

    ということも有り、非常に多く使われています。メジャーなパッケージやフレームワークの導入説明やチュートリアルなどで、yarn での紹介を行っているところも多いです。

    ただ、npm もバージョンアップに伴って、yarn との差はほどんどなくなってきていますので、どちらを使うかは好みの問題と言ってもいいでしょう。

    ここでは、yarn で進めますので、これをインストールします。yarn も npm パッケージとして提供されているので、npm でインストールします。

    1
    sudo npm install -g yarn

    -g は、グローバルにインストールするためのオプションです。ここでのグローバルとは、OSへインストールする、という意味と捉えて問題ありません。

    -g がない場合、実行したディレクトリに node_modules というディレクトリが作成され、その中にパッケージがインストールされます。

    グローバルにインストールする場合、OSへのインストールになるので、sudo が必要になります。

    インストールが完了したら、yarn コマンドでバージョンを確認します。

    1
2
    $ yarn -v
1.22.19

    yarn の最新のメジャーバージョンは 3 なのですが、パッケージのファイルの管理方法が大きく変わり、 そのサイズが小さくなるなどのメリットが有るのですが、これまでのパッケージが参照できなくなり、 調整等が必要になるため、あまり広まっていません。多くは 1 のまま使われています。

    yarn の他にも pnpm というパッケージマネージャもあります。

    まとめ

    Linux 環境に、Node.js を導入することができました。

    Node.js は、特に Web 開発において欠かせないものとなっていますし、 バッチのツールとして使用することで、より便利に使うことができます。

    実際に開発環境としての利用や、パッケージマネジャについては、別途コンテンツを用意していますので、そちらをご確認ください。
