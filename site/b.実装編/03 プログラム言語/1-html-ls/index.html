    HTML Living Standard

    筆者: 海老原 賢次（ebi311@gmail.com）
    作成日: 2022-04-28
    更新履歴:

    • 2022-04-28: 新規作成

    概要

    HTML は、本来ドキュメントに要素(タグ)によってメタ情報を付与することで構造化することが目的でしたが、正式な仕様の前にブラウザの実装が先行することで、見た目に影響するようなものも含まれるなど、混乱が続いていました。

    その後 HTML5 として、原点に立ち戻るとともに、文書の構造化として不適切な要素の削除と、不足していた要素の追加が行われました。合わせて、ブラウザで表示するときの仕様を CSS で決めるのですが、そちらも同時に改定されています。(CSS3 を参照のこと)

    また、Web アプリケーションの UI として利用されることにも配慮しており、フォーム要素にも変更が入っています。

    しかし、HTML の仕様を策定している W3C の動きが遅いことから、Apple、Mozilla、Opera、マイクロソフトによる WHATWG という団体によって、HTMLの定義が策定されることとなりました。

    そのため、HTML5は廃止となり、HTML Living Standard（以下 HTML LS） が業界の標準となっています。

    しかし、2022年現在では、HTML5 と HTML LS とでは大きな差はなく、細かい違いだけで実際の開発にはほとんど影響しません。

    また、HTML LS には、明確なバージョンは無く、日々更新されるようです。Webアプリを作る上では、変更されたり新しい要素をブラウザのサポートがされているかを確認する必要があります。

    ここでは、変更の大きかった HTML4.1 から HTML5 への変更の概要を説明します。具体的な変更点や各要素の説明は、下記を参考にしてください。

    参考

    • W3C: HTML5 における HTML4 からの変更点 - HTML LSとHTML5の違いについてまとめ - Qiita

    THML4.1 → HTML5 で廃止された要素

    主に、文書のメタ情報ではなく、見た目に影響がある要素が削除されました。

    • basefont
    • big
    • center
    • font
    • strike
    • tt

    以上は見た目に関することなので、CSS で定義します。

    • frame
    • frameset
    • noframes

    以上は、iframe や CSS のレイアウトの仕様で定義します。

    • acronym
    • applet
    • isindex
    • dir

    以上は、ほぼ使用されない、また同様のものが別にあり混乱するために削除されました。

    HTML4.1 → HTML5 で廃止された属性

    属性(attribute)も、見た目に関する定義のものを中心に、削除されています。CSS で定義してください。

    参考

    W3C

    完全なリストは、上のMDNのサイトを参照してください。

    • align (全要素から)
    • background (body)
    • bgcolor, cellpadding, cellspaceing, height, with, valign, nowrap (table とその関連要素)
    • type (li, ol, ul などリスト関連)

    文書の構造を表す新しい要素が多数追加されています。

    特徴的なものを挙げておきます。

    dialog

    参考

  btn_open.addEventListener(

    function () {

      dialog.showModal();
    },
    false

    "click",
    function () {

    false

      <button id="close">OK</button>
    </form>
  </dialog>
<body>
</html>

    section

    参考

    <section>: 汎用セクション要素 - HTML: HyperText Markup Language | MDN

    文書の一般的なセクション（関連する情報のまとまり）を表します。h1~6 とともに使用することが想定されています。

    あくまでも、情報としてのまとまりという意味を示すもので、レイアウトのために使用するべきではありません。

    また、適切な見出しが付けられないような内容であれば、<section>を使うべきではないかもしれません。その場合は、<article>を使用することを検討してください。

    header / footer

    <header> - HTML: HyperText Markup Language | MDN

    <footer> - HTML: HyperText Markup Language | MDN

    <header>は、イントロダクションやナビゲーションのグループなどを表します。見出し要素やロゴ、作成者、作成日時などの情報を入れるのに適しています。

    <head>とは目的が全く異なるので、注意してください。

    <footer>は、文書の最後に入れるような、関連文書へのリンク、著作情報などを入れるのに適しています。

    それぞれ、ページ内で 1 つづつのみ使うことが普通ですが、<article>が複数ある、つまり複数の記事がある場合などは複数設けることも考えられます。

    meter, progress

    それぞれ、上限、下限がある範囲のある値で、現在値がどのあたりなのかを示すことが出来ます。ブラウザではバーで視覚的に表示されます。

    値が増減するもの、例えば容量などには<meter>を、進行を表す値、例えばダウンロードやタスクの状態などは、<progress>の利用が適切です。

2
    <meter value="6" max="8" min="0">全８個中６個</meter>
<progress value="40" max="100">進行中です</progress>

    フォーム関連の属性や値の追加

    <input>などのフォームに、多くの属性や typeが追加されました。

    input[type]の値の追加

    <input>の type属性に、多くのものが追加されました。目的に合わせたものを指定することで、ブラウザが適切なキーボードや入力候補を表示します。

    • number : 数値を入力するためのフォーム。対応ブラウザであれば、数値だけが入力できる。
    • tel : 電話番号を入力するためのフォーム。スマホなどでは電話番号のキーボードが表示されます。
    • search : 検索文字列を入力するためのフォーム。
    • url : URL を入力するためのフォーム。
    • email : e-mail アドレスを入力するためのフォーム。
    • data, time ,datetime, datetime-local : 日時を入力するためのフォーム。対応ブラウザであれば、ピッカーが表示される。
    • month, week : 月、または週を選択するためのフォーム。対応ブラウザであれば、ピッカーが表示される。
    • range : 数値を入力するためのフォーム。対応ブラウザであれば、マウスで増減できるバーが表示される。
    • color : 色の値を入力するためのフォーム。対応ブラウザであれば、カラーピッカーが表示される。

    placeholder

    フォームに未入力の場合に表示する文字列のことを、プレースホルダーといいます。

    HTML5 以前は、CSS や JavaScript によってこれを表現してきましたが、HTML のみで表現できるようになりました。

    1
    <input placeholder="入力してください。" />

    入力制限

    必須入力や上限、下限値などを指定することが出来ます。ただし、form submit するとき（submit ボタンで送信する場合）のみ動作します。

    JavaScript で入力値を取得して WebAPI を呼び出す場合は機能しません。ただし、CSS で :invalidを使うことで適切な入力でない場合に、表現を変更することが出来ます。

    • required : フォームの入力が必須であることを示します。
    • max, min : typeが number、datetimeなどの入力の場合に、最小値と、最大値を指定できます。
    • pattern : 正規表現にあった入力しか出来ないように指定します。
    • step : typeが number、datetimeなどの入力の場合に、刻み幅を指定できます。

    ドラッグ＆ドロップ

    HTML でドラッグに関する属性が追加されました。JavaScript と組み合わせることで、ドラッグ＆ドロップの処理が容易になりました。

    JavaScript では、下記の属性を取得でき、DataTransfer オブジェクトでデータを格納し、ドロップ時にそこからデータを取得して処理を行うことが出来ます。

    イベントは、ドラッグ関連のイベントが追加されています。

    属性

    • draggable : trueとすることで、ドラッグ可能な領域であることを示します。
    • dropzone : ドロップできる要素であることを示します。copy、move,linkの値が設定できます。

    イベント

    • dragstart : ドラッグ開始時
    • drag : ドラッグが継続している間
    • dragenter : ドラッグ要素がドロップ要素に入った時
    • dragleave : ドラッグ要素がドロップ要素から出た時
    • dragover : ドラッグ要素がドロップ要素に重なっている間
    • drop : ドロップ時
    • dragend : ドラッグ終了時

    HTML5 → HTML LS の違い

    上でも述べたように、HTML LS は日々更新されるので、確認する必要はありますが、2022年5月時点では主に下記の違いがあります。

    • <a href=""> をクリックしたときに、hrefのページに遷移しつつ、別URLにpingを送信するping属性の追加10
    • <img> の遅延読み込みを行う loading の追加11
    • autofocus が <button>や<input>以外の、全てのフォーカス可能な要素にも適用可能に12
    • <h1>は複数使用可能13（ただし <section>も併用すること）

    合わせて、こちらもご確認ください。
